## 摘要
大型语言模型（LLM）正越来越多地被探索用于临床决策支持，但其在复杂肿瘤治疗规划中的可靠性仍不清楚。我们利用72个真实临床病例（涵盖I至IV期）以及通过非对称信息评分生成（AIRG）生成的1147个病例特定评分（评分生成器可访问真实临床决策，而评估模型无法访问），评估了用于生成乳腺癌治疗推荐的智能体LLM系统。比较了七种流程，包括单LLM基线、工具增强系统以及具有事实核查和自主子智能体生成功能的多智能体架构。表现最佳的配置是使用D&C+SA流程的Claude Opus 4.8，其全局得分为0.594 ± 0.025。工具使用和智能体自主性的增加产生了混合效应，在某些设置中提升了性能，但在其他设置中则降低了性能。性能因临床领域和疾病阶段而异，肿瘤学家主导的错误分析揭示了持续存在的临床相关失误，包括不正确或缺失的推荐、有缺陷的论证、引用错误、过时的声明以及过度自信。这些发现表明，智能体LLM系统能够生成临床相关的乳腺癌推荐，但仍不足以用于无监督的临床使用。

## Abstract
Large language models (LLMs) are increasingly being explored for clinical decision support, but their reliability in complex oncology treatment planning remains unclear. We evaluated agentic LLM systems for breast cancer treatment recommendation generation using 72 real clinical cases across stages I to IV and 1,147 case-specific rubrics generated through Asymmetric Information Rubric Generation (AIRG), in which the rubric generator had access to real clinical decisions unavailable to the evaluated models. Seven pipelines were compared, including single-LLM baselines, tool-augmented systems, and multi-agent architectures with fact checking and autonomous subagent spawning. The best-performing configuration, Claude Opus 4.8 with the D&C+SA pipeline, achieved a global score of 0.594 $\pm$ 0.025. Tool use and increased agent autonomy had mixed effects, improving performance in some settings but degrading it in others. Performance varied by clinical domain and disease stage, and oncologist-led error analysis revealed persistent clinically relevant failures, including incorrect or missing recommendations, flawed justifications, citation errors, outdated claims, and overconfidence. These findings suggest that agentic LLM systems can generate clinically relevant breast cancer recommendations, but remain insufficient for unsupervised clinical use.

## 二、论文提出的方法论：核心思想、关键技术细节

- **核心思想**：使用**真实临床病例**和**不对称信息评分生成（AIRG）** 对智能体LLM系统进行基准测试。评分生成器拥有特权信息（真实临床决策），而评估模型仅有病例摘要，从而使评分更贴近临床实际。

- **关键技术细节**：
  1. **数据集构建**：从巴西一家私人肿瘤诊所的电子健康记录（EHR）中收集72例乳腺癌病例（I-IV期各18例），由医学生和认证医师创建结构化病例摘要（含病理、生物标志物、患者评分）及临床决策。
  2. **AIRG评分生成**：  
     - 使用 frontier LLM（gemini-3.5-flash）并配置高推理努力，输入病例摘要、真实临床决策以及网络搜索/PubMed搜索工具。  
     - 生成病例特定评分标准（rubric），每条标准包含一个属性（可存在或缺失）和分值（-10到+10），按四类分组：系统治疗、放疗、手术、补充检查。共生成1147条评分。  
  3. **七种推荐生成流水线**：  
     - **基线**：单个LLM，无工具。  
     - **LLM+网络搜索（WS）**：单个LLM可访问网络搜索。  
     - **LLM+PubMed搜索（PM）**：单个LLM可访问PubMed搜索。  
     - **LLM+PubMed+PMC全文（PM+PMC）**：可检索PMC全文。  
     - **分而治之（D&C）**：一个主协调器+四个预定义子智能体（系统治疗、放疗、手术、补充检查），每个子智能体有权使用所有工具。  
     - **D&C+事实核查（D&C+FC）**：生成初始推荐后，由事实核查智能体（同样有权使用所有工具）检查事实声明和不一致后再输出最终版本。  
     - **D&C+自主子智能体生成+事实核查（D&C+SA）**：不再预定义子智能体，而是由主智能体自主生成子智能体（含定制指令），最终也经过事实核查。  
  4. **评分公式**：  
     - 对每个病例 \(i\)，有 \(M_i\) 条评分标准，每条标准有分值 \(p_{ij}\)，由评分模型判断是否满足（\(I_{ij}=1\) 或0）。  
     - 病例得分：\(s_i = \frac{\sum_j I_{ij} p_{ij}}{\sum_j \max(0, p_{ij})}\)。  
     - 全局得分：\(S = \text{clip}\left( \frac{1}{N}\sum_i s_i, 0, 1 \right)\)。  
     - 通过非参数自助法计算不确定性。

- **工具实现**：使用 Model Context Protocol (MCP) 服务器，网络搜索由 Parallel.ai 提供，PubMed搜索通过官方API和开源MCP服务器。

## 八、不足与局限

1. **数据局限性**：仅来自巴西一家私人诊所，可能不反映不同资源环境、医疗体系和临床实践模式。参考标准为单个医生或MDT决策，缺乏独立共识验证。

3. **LLM-as-a-Judge误差**：评分模型判断可能存在偏差，尤其在边缘病例中。  
4. **模型覆盖不全**：因成本限制，OpenAI和Anthropic模型未在所有流水线上测试，导致比较不完全均衡。  
5. **缺乏鲁棒性测试**：未评估提示变化、信息缺失或干扰对输出的影响。论文指出这可能显著改变结果。

8. **临床部署风险**：尽管得分较高，但错误仍在关键方面（如错误推荐、过度自信）可能造成患者危害，系统尚不能独立使用。
